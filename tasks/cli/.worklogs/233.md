## 2026-03-01T11:00:00Z

Added dependency expansion to `--scope` and a `--scope-exact` flag.

**Problem:** When a task touches "web" but depends on a task with no `touches` field,
`--scope web` wouldn't show the blocking dependency — even though that's the task
you'd need to work on to make progress on the "web" scope.

**Approach:** By default, `--scope` now expands via dependency components (same
approach as the `tracks` command's `assignScope()`). It finds seed tasks touching the
scope, identifies their connected dependency components, then includes any actionable
task in those components. Added `--scope-exact` / `ScopeExact` option to get the old
exact-match-only behavior when needed.

**Changes:**
- `sdk/go/next/next.go`: Added `ScopeExact bool` to `Options`, added `filterByScopeExpanded()` with `buildDepComponents()` (BFS connected components), branching logic in `filterActionable`
- `apps/cli/internal/cli/next.go`: Added `nextScopeExact` flag, `--scope-exact` registration, passed to options
- `sdk/go/next/next_test.go`: Added 2 tests (expansion includes blocking deps, exact skips expansion)
- `apps/cli/internal/cli/next_test.go`: Added 2 tests with dedicated `createScopeDepTestTaskFiles` helper
- `apps/cli/internal/e2e/next_test.go`: Added `TestNext_ScopeExpansion` with 2 subtests

**All tests pass:** unit, integration, e2e, and lint.
